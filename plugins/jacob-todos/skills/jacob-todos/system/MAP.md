# MAP.md — `<workspace>/to do/`

Single source of truth for what's in this folder. Read this first before assuming any file is canonical.

## Canonical v3 system (at root)

| File | Role |
|---|---|
| `cowork_instructions.md` | How the to-do system runs (peculiar conventions, calendar gestures, approval workflow). Read first. |
| `gcal_todo_instructions.md` | Calendar query rules; read before any Google Calendar operation. |
| `state_v3.json` | System state — last runs, thresholds, stacking_day, pruning rules, **calendar configuration**. schema_version: 3. |
| `scripts/tasks_v3.json` | Single source of truth for tasks. Lives inside `scripts/` because all v3 generators read it via `__dirname`. |
| `check_in.html` | Interactive review form — primary decision surface. Each task with a solution links to its per-task recommendation page under `recs/`. Generated by `scripts/build_check_in.js`. |
| `recs/rec_<task_id>.html` | Per-task recommendation pages (one HTML file per task). Rich-HTML rendering (headers/lists/tables/links via `marked`). Linked from `check_in.html` via the "View suggested plan →" button. Generated by `scripts/build_per_task_recs.js`. |
| `done.md` | Closed-task archive (plain text). |
| `someday_list.md` | Low-urgency parking (plain text, hand-edited). |
| `inbox.md` | Open-questions intake (read fresh on each consolidation). |

## Rolling outputs (auto-pruned per `state_v3.json.pruning_rules`)

- `check_in_YYYY_MM_DD.md` — last N check-in reports (default 10).

## Subfolders

- `scripts/` — generators, the verb library (`task_verbs.js`), `pickup_actions.js`, plus the live `tasks_v3.json`.
- `recs/` — per-task recommendation HTML pages (one per active task). Regenerated by `build_per_task_recs.js`; linked from `check_in.html`. Safe to delete — fully regenerated each run.
- `archive/` — auto-rotated snapshots and pre-slim doc backups (created lazily as pruning kicks in).

## What NOT to open from the root

Anything in `archive/` is historical record, not the live system. If a session offers to open something under `archive/` as if it were live — that's a stale suggestion.
